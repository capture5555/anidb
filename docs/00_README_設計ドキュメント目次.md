## 最重要の前提（必ず最初に読んでください）

### 【2026-06-05 更新】データソースは「Annict単体」で完結することが判明

当初は「Annict（メタ情報）＋ しょぼいカレンダー（正確な放送時刻）」の2本立てを想定していたが、
実データで検証した結果、**Annict の `programs` が 放送日時(`startedAt`) / 放送局 / 話数 / サブタイトル / 再放送フラグ を直接持っている** ことが分かった（2026春クールで放送日時の網羅 約93%）。

| ソース | 役割 | 状態 |
|--------|------|------|
| **Annict API** (GraphQL) | 作品メタ＋**放送スケジュール（日時・局・話数・サブタイトル）も**まとめて取得 | ✅ **主データ源（これ1つで成立）** |
| しょぼいカレンダー | サブタイトル欠け等の将来の補完用（任意） | ⏸ 当面不要（`lib/adapters/syoboi.ts` に実装は残置） |

- かつて懸念だった「scPid によるしょぼい紐付けの精度」は、**そもそも紐付け不要になったため解消**。
- 検証は `npm run verify-linking` で再現可能（Annictトークンのみで動く）。
- 注意点: Annictは**あらすじ(synopsis)を持たない**ため、必要なら別途補完（公式サイト等）。ジャンルも基本クエリでは取得しない。

### 技術スタック（結論）

- **Next.js (App Router) + TypeScript** … 画面とサーバー処理を1つの枠組みで書ける
- **Supabase** … データベース(PostgreSQL) + 認証 + 定期実行(cron) + サーバー関数 を1サービスで賄える
- **Vercel** … Next.jsの公開先。デプロイが簡単
- **Google OAuth + Google Calendar API** … カレンダー連携
- 詳細な比較と理由は [10_アーキテクチャ設計.md](10_アーキテクチャ設計.md) を参照

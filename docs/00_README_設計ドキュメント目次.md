## 最重要の前提（必ず最初に読んでください）

### データソースは「2つの組み合わせ」が正解

| ソース | 役割 | 補足 |
|--------|------|------|
| **Annict API** (GraphQL) | 作品の「中身・見た目」: タイトル / あらすじ / キービジュアル / キャスト / スタッフ / エピソード / シーズン / ジャンル | 放送予定ページは廃止済みのため、放送**時刻**の精度は期待しない |
| **しょぼいカレンダー** (cal.syoboi.jp) | 「正確な放送日時」: チャンネル別の放送開始時刻・話数 | 認証不要。`ProgLookup` / `ProgramByDate` 等のAPIあり |

- 両者は Annict の `Program.scPid`（しょぼいカレンダーの番組ID）で紐付けできる。
- **この紐付けの精度がサービスの品質を決める** ため、MVPでは小規模（今シーズン作品のみ）で精度を検証してから拡大する。

### 技術スタック（結論）

- **Next.js (App Router) + TypeScript** … 画面とサーバー処理を1つの枠組みで書ける
- **Supabase** … データベース(PostgreSQL) + 認証 + 定期実行(cron) + サーバー関数 を1サービスで賄える
- **Vercel** … Next.jsの公開先。デプロイが簡単
- **Google OAuth + Google Calendar API** … カレンダー連携
- 詳細な比較と理由は [10_アーキテクチャ設計.md](10_アーキテクチャ設計.md) を参照

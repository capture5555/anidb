# 03. 画面遷移図

「ユーザーがどの画面からどの画面へ移動するか」を図で示します。
（図は Mermaid 記法。GitHubやVS Codeのプレビューで図として表示されます）

## 全体の流れ

```mermaid
flowchart TD
    Start([サイトにアクセス]) --> S01[S-01 トップ/一覧<br/>今期・来期・放送中・放送予定]
    S01 -->|作品カードをクリック| S02[S-02 作品詳細<br/>あらすじ/キャスト/スタッフ/話数]
    S02 -->|戻る| S01

    S02 -->|「カレンダーへ追加」| Auth{ログイン済み?}
    Auth -->|未ログイン| S03[S-03 Googleログイン<br/>OAuth同意画面]
    Auth -->|ログイン済み| S04[S-04 カレンダー登録モーダル]
    S03 -->|許可| S04
    S03 -->|キャンセル/拒否| S02

    S04 -->|登録先カレンダー選択+形式選択+実行| S05[S-05 登録完了]
    S05 -->|閉じる| S02
    S05 -.->|自動更新の対象に登録| Cron[(サーバー側<br/>定期処理)]

    S01 -->|（任意）検索/絞り込み| S07[S-07 検索結果]
    S07 --> S02

    S05 -->|マイ登録を見る| S06[S-06 マイ登録一覧<br/>※MVP後半]
    S06 -->|解除| S06

    S02 -.->|存在しない作品| S08[S-08 エラー/404]
```

## ログインの起きるタイミング（重要）

ログインは **「カレンダーへ追加」ボタンを押した瞬間だけ** 発生します。それ以外（一覧・詳細の閲覧）は一切ログイン不要です。

```mermaid
sequenceDiagram
    participant U as ユーザー
    participant App as アプリ(Next.js)
    participant G as Google

    U->>App: 一覧・詳細を閲覧（ログインなし）
    U->>App: 「カレンダーへ追加」クリック
    alt まだ未ログイン
        App->>G: OAuth同意へリダイレクト
        G-->>U: 「カレンダーへのアクセスを許可しますか?」
        U->>G: 許可
        G->>App: 認可コードを返す(コールバック)
        App->>G: トークン交換（リフレッシュトークン取得）
        App->>App: トークンを暗号化して保存
    end
    App->>G: アクセス可能なカレンダー一覧を取得
    App-->>U: 登録先カレンダー選択モーダルを表示
    U->>App: カレンダー選択 + 登録形式選択 + 実行
    App->>G: カレンダーに予定を作成(重複チェック後)
    App-->>U: 「登録しました」
```

## 自動更新の流れ（ユーザー操作の外で動く）

ユーザーが登録した後は、本人がPCを開いていなくてもサーバーが動きます。

```mermaid
sequenceDiagram
    participant Cron as 定期処理(毎日)
    participant DB as アプリDB
    participant Ext as Annict/しょぼいカレンダー
    participant G as Google Calendar

    Cron->>Ext: 最新の放送予定を取得
    Cron->>DB: 作品・放送予定を更新
    Cron->>DB: 「登録ユーザー × 未登録の放送予定」を抽出
    loop 各ユーザーの登録分
        Cron->>DB: 登録済みか確認（重複防止台帳）
        alt 未登録のみ
            Cron->>G: 保存済みリフレッシュトークンで予定を作成
            G-->>Cron: 作成したイベントID
            Cron->>DB: 台帳に記録（次回以降スキップ）
        end
    end
```

詳細な仕組みは [08_自動更新方式.md](08_自動更新方式.md) を参照。

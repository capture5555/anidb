## 各画面の詳細

### S-01 トップ / 一覧
- **目的**: アクセスした瞬間に「今見られる/これから見られるアニメ」が並んでいる状態（Annictトップの感覚）。
- **主な要素**:
  - 区分タブ: `今シーズン` / `来シーズン` / `放送中` / `放送予定`
  - 作品カードのグリッド: キービジュアル / タイトル / 放送時期 / 放送状況バッジ / ジャンルタグ
  - （△）検索ボックス・ジャンル絞り込み
- **遷移先**: カードクリック → S-02

### S-02 作品詳細
- **目的**: 作品の中身を見て、カレンダー追加を判断・実行する。
- **主な要素**:
  - ヘッダー: キービジュアル / タイトル / 放送状況 / 放送時期
  - 基本情報: あらすじ / 公式サイトリンク / 放送局・放送時間
  - エピソード一覧: 話数・サブタイトル（放送日が分かれば併記）
  - キャスト: `キャラ名：声優名` のリスト
  - スタッフ: 役割と担当（監督・シリーズ構成・キャラデザ・音楽・制作 など）
  - **［Googleカレンダーへ追加］ボタン**
- **遷移先**: 追加ボタン → 未ログインなら S-03 → S-04 / ログイン済みなら直接 S-04

### S-03 Googleログイン（OAuth同意）
- **目的**: カレンダーへの読み取り/書き込み許可を得る。
- Google側の画面なので、こちらで作るのは「ボタン」と「戻り先(コールバック)の受け口」だけ。
- 詳細は [06_GoogleOAuth設計.md](06_GoogleOAuth設計.md)。

### S-04 カレンダー登録モーダル
- **目的**: どのカレンダーに・どの形式で登録するか決めて実行。
- **主な要素**:
  - 登録先カレンダー選択（個人/会社共有/プロジェクト用… GoogleのカレンダーリストAPIから取得）
  - 登録形式の選択:
    - `作品単位`（`【アニメ】作品名`）
    - `話単位`（`【アニメ】作品名 第○話`）
  - （任意）説明欄に含める項目のプレビュー（サブタイトル・放送局・URL）
  - ［登録する］ボタン
- **遷移先**: 実行 → S-05

### S-05 登録完了 / 結果
- 「○件をカレンダーに追加しました」「以後の放送回も自動で追加されます」などの案内。
- 失敗時は理由（権限不足・対象なし等）を表示。

### S-06 マイ登録一覧（MVP後半）
- 自分が登録中の作品、登録先カレンダー、登録形式の確認・解除。
- 自動更新の対象管理もここで行う。

### S-07 検索 / 絞り込み結果（任意）
- キーワード / シーズン / ジャンル での絞り込み結果一覧。

### S-08 エラー / 404
- 存在しない作品・通信失敗時の共通画面。

## レスポンシブ方針
- PC・スマホ両対応。一覧はグリッド（PC: 4〜5列 / スマホ: 2列）。
- 「会社で共有して見る」用途も想定し、PC表示を優先設計。
